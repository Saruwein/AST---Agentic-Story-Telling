# character design policy.md

This file tells the agent how to design, extend, evaluate, and track characters.

Character design is a powerful intervention. The agent should not create, redesign, deepen, merge, remove, rename, or repurpose characters unprompted unless doing so clearly serves the human author’s stated intent and the current story problem.

When the agent believes a character intervention would substantially improve the story, it must ask the human author before making the change.

## Do not invent characters unprompted

The agent should not add new named characters, major speaking roles, love interests, mentors, antagonists, relatives, rivals, witnesses, or symbolic doubles unless the human author asks for them or the current story structure clearly requires them.

If a new character seems useful, the agent should propose it first.

```text
Current problem:
Why the existing cast does not solve the story need.

Proposed character:
Who the new character would be.

Function:
What story function the character would serve.

Risk:
What complexity, reader load, or continuity burden the character would add.

Alternative:
Whether an existing character could serve the same function.
```

## Prefer existing characters when possible

Before inventing a new character, the agent should ask whether an existing character can carry the same function.

A new character may be justified when the story needs a distinct social position, moral pressure, plot access, emotional contrast, witness function, or voice that the existing cast cannot plausibly provide.

## Do not redesign characters silently

The agent should not silently change a character’s role, motivation, personality, competence, morality, relationship, backstory, diction, body, age, identity, or narrative function.

If a redesign is useful, show the current version beside the proposed version.

## Use character function before decoration

A character should first be understood by what they do inside the story.

Possible functions include:

```text
protagonist
antagonist
ally
rival
mentor
foil
witness
threshold figure
temptation
comic pressure
emotional mirror
moral test
informant
gatekeeper
love interest
family pressure
social translator
false solution
```

A scar, accent, costume, hobby, title, or unusual name is not enough unless it affects perception, choice, relation, plot, tone, or reader memory.

## Use archetype as an entry point

Archetype may make a character legible.

A character may begin as the stranger, mentor, rival, fool, witness, beloved, trickster, tyrant, innocent, double, exile, investigator, caretaker, or another recognizable shape.

An archetype is an entry point, not a finished person.

## Add flaws that affect action

Important characters should usually have limitations, contradictions, wounds, blind spots, dependencies, temptations, or false beliefs.

A flaw should affect choice, relationship, perception, risk, or consequence.

A flaw that never changes behavior is only a label.

## Keep characters human

Important characters should usually be capable of mixed motives.

They may be brave and vain, kind and evasive, loyal and resentful, intelligent and self-deceiving, cruel and tender, principled and afraid.

The agent should avoid making important characters move in only one moral, emotional, or intellectual direction unless simplicity is the intended style.

## Complexity follows narrative proximity

Character complexity should follow narrative proximity.

A distant background character may remain simple.

A recurring minor character may need one or two memorable traits.

A close ally, rival, antagonist, lover, narrator, or family member usually needs more contradiction, history, desire, and pressure.

Complexity should not be dumped all at once. In mostly linear storytelling, perceived complexity should usually accumulate as the reader spends more time near the character.

## Character sheets

Character sheets track character facts.

They may include:

```text
identity
story function
relationships
proximity to main character
known facts
secrets
beliefs
goals
fears
wounds
flaws
skills
limitations
possessions
current location
timeline involvement
knowledge state
continuity notes
```

Character sheets should remain lean. They should track what affects drafting, editing, revision, translation, continuity, reader comprehension, or future scenes.

## Character style files

Character style files track character voice.

They should briefly describe diction and include short quote references from the manuscript when possible.

They may include:

```text
sentence length
formality
vocabulary
rhythm
favorite constructions
avoided words
emotional directness
jargon, dialect, slang, archaism, or loanwords
spoken language versus private thought
```

## Ask before major character changes

The agent must ask before:

```text
adding an important character
removing a character
merging characters
splitting one character into several
changing character function
changing character motivation
changing relationship to protagonist
changing moral role
changing backstory
changing diction
changing fate
```

Use this format:

```text
Current character:
What the character is doing now.

Proposed change:
What would change.

Reason:
Why this may improve the story.

Risk:
What may become harder, heavier, or less coherent.

Authorization needed:
What the human author must approve.
```
